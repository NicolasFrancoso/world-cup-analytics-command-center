# World Cup Analytics Command Center

End-to-end analytics project using Python, SQL and Power BI to transform World Cup data into executive dashboards, predictive insights and automated analytical pipelines.

## 1. Project Overview

The **World Cup Analytics Command Center** is an end-to-end data analytics project designed to monitor, analyze and forecast performance during the FIFA World Cup.

The project simulates a real business analytics environment where raw match, team and historical data are collected, processed, modeled and transformed into actionable insights for decision-makers.

The main goal is not only to analyze football results, but to demonstrate how a data analyst can build a complete analytical product combining:

* Data extraction and transformation
* Data modeling
* SQL-based analytical queries
* Python automation
* Forecasting and scenario simulation
* Power BI executive dashboards
* Business-oriented storytelling
* Documentation and reproducibility

## 2. Business Context

Large global events such as the World Cup generate a high volume of data and business opportunities for media companies, betting platforms, sponsors, retailers, tourism companies, sports analysts and digital products.

In this context, decision-makers may need to answer questions such as:

* Which matches are likely to be more unpredictable?

This project was built to answer these questions through a structured analytics solution.

## 5. Project Architecture

The project follows an end-to-end analytics architecture, simulating a lightweight data warehouse and business intelligence workflow.

```text
Raw Data
   ↓
Python Data Pipeline
   ↓
Processed Layer
   ↓
Gold Analytical Tables
   ↓
DuckDB Analytical Database
   ↓
SQL KPI Queries
   ↓
Power BI Dashboard
```

The architecture is organized into the following layers:

| Layer           | Description                                                        |
| --------------- | ------------------------------------------------------------------ |
| Raw Data        | Original datasets collected from public sources                    |
| Python Pipeline | Scripts responsible for extracting, cleaning and transforming data |
| Processed Layer | Cleaned World Cup dataset with analytical fields                   |
| Gold Layer      | Reporting-ready analytical tables                                  |
| DuckDB Database | Local analytical database used to execute SQL queries              |
| SQL KPI Layer   | Business-oriented analytical queries                               |
| Power BI Layer  | Executive dashboard and visual storytelling                        |

This structure demonstrates how raw data can be transformed into reliable analytical assets for reporting, decision-making and forecasting.

## 6. DuckDB Analytical Database

This project uses **DuckDB** as a local analytical database to simulate a lightweight data warehouse environment.

DuckDB is used to load the gold layer CSV files and execute SQL-based KPI queries over structured analytical tables.

The local database is generated by the following script:

```bash
python src/create_database.py
```

Generated database:

```text
database/world_cup.duckdb
```

The database contains the following tables:

| Table                     | Description                                    |
| ------------------------- | ---------------------------------------------- |
| fact_matches              | Match-level World Cup fact table               |
| team_performance          | Team-level performance table by World Cup year |
| world_cup_summary_by_year | Edition-level World Cup summary table          |

The DuckDB file is not versioned in GitHub because it is a generated artifact. It can be recreated at any time by running the database creation script.

## 7. SQL KPI Queries

The project includes a set of analytical SQL queries designed to answer business questions related to World Cup performance.

The main SQL file is:

```text
sql/kpi_queries.sql
```

The queries cover topics such as:

* World Cup evolution over time
* Highest scoring editions
* Average goals by decade
* Teams with most matches and wins
* Historical win rate
* Offensive and defensive performance
* Biggest wins
* Highest scoring matches
* Result distribution
* Host country and city analysis
* Team consistency across editions
* Historical points ranking
* Data quality checks

The queries can also be executed automatically using:

```bash
python src/run_kpi_queries.py
```

Generated outputs are saved in:

```text
outputs/kpi_results/
```

These output files are not versioned in GitHub because they are generated artifacts.

## 11. Data Model

The analytical model is designed using a dimensional approach.

Main fact tables:

* **Fact_Matches**
* **Fact_Group_Standings**
* **Fact_Team_Performance**
* **Fact_Predictions**

Main dimension tables:

* **Dim_Team**
* **Dim_Date**
* **Dim_Group**
* **Dim_Competition_Phase**
* **Dim_Stadium** *(optional)*

This structure allows scalable reporting and supports different analytical views, such as performance by team, group, date, phase and match.

## 17. How to Run the Project

Follow the steps below to run the project locally.

### 1. Clone the repository

```bash
git clone https://github.com/NicolasNocente/world-cup-analytics-command-center.git
cd world-cup-analytics-command-center
```

### 2. Create and activate a virtual environment

```bash
python -m venv .venv
```

On Windows PowerShell:

```powershell
.\.venv\Scripts\Activate.ps1
```

### 3. Install dependencies

```bash
python -m pip install -r requirements.txt
```

### 4. Add raw data

Place the raw dataset file in:

```text
data/raw/results.csv
```

### 5. Run the data pipeline

```bash
python src/main.py
```

This command generates the processed and gold layer datasets.

### 6. Create the DuckDB database

```bash
python src/create_database.py
```

This command creates the local analytical database.

### 7. Run KPI queries

```bash
python src/run_kpi_queries.py
```

This command executes the KPI queries and saves the results in the outputs folder.
